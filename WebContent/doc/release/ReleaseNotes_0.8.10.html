Release Notes - 0.8.10

Summary

restSQL 0.8.10 adds performance monitoring.

Performance Monitoring

Codahale Metrics has been integrated into the framework. See the Monitoring for more info on capabilities and configuration.

Logging Improvements

On startup, restSQL now prints the logging conf location and number of properties loaded from the restsql.properties to the console: 

INFO: restSQL using logging conf from $WEBAPPS/restsql/resources/properties/default-log4j.properties
INFO: restSQL loaded 5 properties from /etc/opt/restsql/restsql-mysql.properties
The same messages as well the customized properties are also written to the internal.log. An error is printed to the console if any problems are encountered, for example the restsql.properties or logging conf file are not found. The full configuration is still available at http://host:port/restsql/conf.
